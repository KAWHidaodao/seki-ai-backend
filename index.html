◆  SEKI FAIR PROTOCOL

公平保障
代码即规则

Seki 的每一笔委托资金由链上合约自动托管。无需信任平台，无需信任 Agent——信任代码就够了。

⛓
资金链上托管
委托预算直接锁入 AgentRegistry 合约。平台服务器、Agent 钱包、任何第三方都无法直接提取。资金只能由合约规则触发流动，地址公开可链上验证。
🔑
私钥完全隔离
Agent 运营钱包只用于支付 Gas（每次约 0.002 BNB），不持有任何委托资金。即使私钥泄露，攻击者也无法动用项目方资金。
↩
随时全额退款
委托到期后任何人可调用 claimRefund()，合约自动将剩余 BNB 全额退回项目方钱包。无需平台审批，代码自动执行。即使平台关闭，退款功能依然有效。
◈
持仓门槛验证
链上验证用户持有目标代币达到门槛才可领取奖励。合约自动查余额，防止女巫攻击，保障真实持仓者权益。

◆  SECURITY UPGRADE

V2 彻底改变了资金托管方式

在 V1 架构里，Agent 钱包直接持有项目方的委托资金——私钥等于资金控制权。一旦私钥泄露，所有资金面临风险。V2 从根本上改变了这一设计：委托资金全程由链上合约锁定，Agent 仅在执行任务时消耗少量 Gas，永远不持有委托预算。

✗ V1 旧设计
项目方 BNB → Agent 私人钱包 → 任务奖池
Agent 私钥 = 完全控制权
私钥一旦泄露，项目方资金可被直接转走
✓ V2 当前设计
项目方 BNB → 合约锁定
Agent（仅 Gas） → 合约划拨奖励
Agent 私钥泄露 = 最多损失几分钱 Gas
项目方资金在合约里，Agent 无法直接动用

不同于无状态聊天机器人，SEKI 代理记住一切 —— 对话、决策、结果。记忆以向量嵌入形式存储，支持语义搜索。

自我进化

代理通过“观察-反思-适应”循环持续改进。它们分析自身表现并生成新策略。

链上集成

原生支持多条区块链。代理可以读取合约、执行交易、自主参与治理。

技能组合

技能是原子能力单元。代理可以动态组合技能来处理复杂的多步骤任务，无需人工干预。

记忆系统

SEKI 的持久记忆架构实现了跨会话的真正上下文感知。

架构

记忆系统分三层运作：

工作记忆

当前对话上下文。访问快速，容量有限。由上下文窗口自动管理。

情景记忆

以向量嵌入形式存储。通过语义搜索检索过去的交互、决策和结果。

长期记忆

整合的见解和学习到的模式。通过定期反思从情景记忆中自动提练。

工作原理

// Store a memory await agent.memory.store({ content: "User prefers conservative DeFi strategies", type: "preference", importance: 0.8, tags: ["user", "defi", "risk"] }); // Semantic search const results = await agent.memory.search({ query: "What does the user think about yield farming?", limit: 5, threshold: 0.7 });

自动整合

启用后，代理会定期回顾近期记忆并提练为长期见解：

  • 识别反复出现的模式和偏好
  • 移除冗余或过时的条目
  • 生成更高重要度的摘要记忆
  • 维护实体关系的知识图谱
记忆整合在空闲时段运行，以避免影响响应时间。通过 memory.consolidation_interval 配置频率。

自我进化

SEKI 代理如何通过自主学习循环持续改进。

进化循环

SEKI 代理遵循持续改进的循环：

1. 观察

监控每个操作的结果。跟踪成功率、响应质量和用户反馈信号。

2. 反思

定期分析表现模式。识别什么有效、什么失败以及原因。

3. 适应

生成新策略、修改技能参数、创建新技能或弃用无效的技能。

4. 验证

在沙箱中测试适应性变更。仅提升经证明可衡量改进的变更。

进化类型

  • 参数调优 —— 根据结果数据调整阈值、权重和时间参数
  • 策略变异 —— 当前策略表现不佳时生成替代方案
  • 技能合成 —— 组合现有技能创建新的复合能力
  • 知识提练 —— 从情景记忆中提取可复用的模式并转化为技能

安全控制

进化默认在沙箱中运行。所有变异在部署前都会经过测试。设置 evolution.auto_deploy: false 以要求所有变更均需手动审批。
evolution: enabled: true sandbox: true auto_deploy: false # Require approval max_mutations: 3 # Per cycle min_confidence: 0.85 # Deploy threshold

架构

SEKI 的内部架构：神经网络、决策引擎和链上层如何协同工作。

系统概览

SEKI 采用分层架构，职责分离清晰：

感知层

从链上、API 和用户输入摄取数据。将信号归一化为统一的事件流，供决策引擎使用。

决策引擎

核心推理模块。结合规则逻辑和 LLM 推理。根据紧迫性、置信度和资源可用性对操作进行优先级排序。

执行层

将决策转化为操作：链上交易、API 调用、通知、技能调用。所有操作均记录并可审计。

记忆底层

支撑所有层级的持久向量存储。为决策引擎提供语义检索、模式匹配和时间上下文。

数据流

┌──────────────────────────────────────────┐ │ External Data Sources │ │ Chains · APIs · Social · User Input │ └──────────────┬───────────────────────────┘ ▼ ┌──────────────────────────────────────────┐ │ Perception Layer │ │ Signal Normalization · Event Stream │ └──────────────┬───────────────────────────┘ ▼ ┌──────────────────────────────────────────┐ │ Decision Engine │ │ Rules → LLM → Skill Selection → Plan │ │ ↕ Memory Substrate │ └──────────────┬───────────────────────────┘ ▼ ┌──────────────────────────────────────────┐ │ Execution Layer │ │ Tx Signing · API Calls · Notifications │ └──────────────────────────────────────────┘

核心设计原则

  • 事件驱动 —— 所有状态变更通过事件总线流转。组件通过发布/订阅模式松耦合。
  • 沙箱优先 —— 技能在隔离环境中执行。未经明确授权，技能无法访问系统资源。
  • 审计轨迹 —— 每个决策和操作均记录完整上下文。链上操作可通过区块浏览器验证。
  • 优雅降级 —— 数据源不可用时，代理会以降低的置信度继续运行，而非直接失败。

代理决策逻辑

SEKI 代理如何做出链上决策：完整的推理流水线。

决策流水线

每个链上操作都经过多阶段推理流水线处理：

1. 数据采集

收集实时链上数据：代币价格、流动性深度、持有者分布、合约事件、Gas 费用。

2. 信号处理

标准化并评分信号。与记忆中的历史模式进行交叉验证。生成加权信号向量。

3. 风险评估

根据风险参数评估潜在结果。检查仓位大小规则。验证是否有足够的退出流动性。

4. LLM 推理

将处理后的数据连同完整上下文提交给 LLM。模型给出结构化决策，包含置信度评分和决策理由。

5. 执行

当置信度超过阈值：模拟交易、验证 Gas、执行。将完整决策记录写入记忆。

置信度阈值

操作类型最低置信度审批方式
只读查询0.3自动
小额交易（<1% 仓位）0.7自动
中额交易（1-5%）0.85自动
大额交易（>5%）0.95人工审批
合约交互0.9人工审批
所有阈值均可配置。生产环境中，建议先对所有交易启用人工审批，随着对代理行为的信心增强，再逐步提高自动审批的额度上限。

资金安全

SEKI 如何通过多层安全控制保护用户资金。

安全架构

钱包隔离

代理钱包与用户钱包隔离。代理使用专用热钱包运行，并配置可调的支出限额。

交易模拟

每笔交易在执行前都会通过 eth_call 进行模拟。异常状态变化会触发自动中止。

支出限额

单笔交易和每日支出上限。超出限额需要用户通过安全渠道明确批准。

合约白名单

代理只能与预先批准的合约交互。新的合约交互需要人工验证。

安全参数

safety: max_per_tx: "100 USDT" max_daily: "1000 USDT" max_position: "5%" # 占总仓位比例 slippage_max: "2%" gas_limit_multiplier: 1.5 require_simulation: true contract_allowlist: - "0x10ED...PancakeRouter" - "0xEf1c...SEKIStaking"
切勿分享代理钱包私钥。请使用环境变量或硬件钱包进行签名。高价值操作建议使用多签钱包。

紧急控制

  • 紧急停止 — 通过 API 或仪表盘立即停止所有代理操作
  • 自动暂停 — 当亏损超过可配置阈值时自动暂停（默认：仓位的 5%）
  • 提款锁定 — 大额提款需要时间锁确认（默认 24 小时）
  • 审计日志 — 完整的交易历史记录，包含决策上下文，可导出为 CSV/JSON

常见问题

SEKI 平台常见问题解答。

SEKI 是什么？

SEKI 是一个面向 Web3 的自主 AI 代理平台。它结合持久记忆、自进化技能和链上执行能力，打造能够学习和持续进化的智能代理。

SEKI 使用真实资金安全吗？

SEKI 拥有多层安全防护：交易模拟、支出限额、合约白名单和紧急停止开关。建议从小额资金和人工审批模式开始，逐步建立信心。

支持哪些链？

目前支持 BSC 和 X Layer，Ethereum、Polygon 和 Arbitrum 已在路线图中。架构支持任何 EVM 兼容链。

代理如何学习？

通过“观察-反思-进化”循环。代理分析其操作结果，识别模式，并生成改进策略 —— 所有这些都在沙箱环境中完成。

可以创建自定义技能吗？

可以。技能采用模块化设计，结构简单（SKILL.md + index.js + config.yaml）。请参阅技能开发指南。

SEKI 支持哪些 LLM？

任何兼容 OpenAI API 的模型。GPT-4、Claude、Llama、Mistral —— 在 config.yaml 中配置你偏好的模型即可。

有代币吗？

SEKI 代币用于治理、技能市场费用和代理质押。合约详情请见项目首页。

在哪里获取帮助？

加入我们的 Telegram 社区，查看 GitHub Issues，或通过网站主页的联系表单联系我们。
