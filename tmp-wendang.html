Core Concepts

Persistent Memory

Unlike stateless chatbots, SEKI agents remember everything — conversations, decisions, outcomes. Memory is stored as vector embeddings with semantic search.

Self-Evolution

Agents continuously improve through an observe-reflect-adapt loop. They analyze their own performance and generate new strategies.

On-Chain Integration

Native support for multiple blockchains. Agents can read contracts, execute transactions, and participate in governance autonomously.

Skill Composition

Skills are atomic capability units. Agents can combine skills dynamically to handle complex, multi-step tasks without human intervention.

Quick Start

Get your first SEKI agent running and connected to the blockchain.

1. Install SEKI

# Install via npm npm install -g @seki/cli # Or clone from source git clone https://github.com/seki-ai/seki.git cd seki && npm install

2. Initialize Agent

# Create a new agent workspace seki init my-agent # Configure your API keys cd my-agent cp .env.example .env # Edit .env with your keys

3. Start the Agent

# Launch in development mode seki start --dev # Agent will initialize memory, load skills, # and connect to configured chains ✓ Memory system initialized ✓ Loaded 12 skills ✓ Connected to BSC mainnet ✓ Agent ready at http://localhost:3000
The agent starts with a default skill set. It will learn and evolve new skills based on the tasks you give it.

4. First Interaction

Open the agent dashboard and try these commands:

// Check agent status > status // Ask it to analyze a token > analyze 0x1234...abcd on BSC // Set up monitoring > watch wallet 0xdead...beef for large transfers

5. Connect to Chain

Add blockchain connections in your configuration:

# config.yaml chains: bsc: rpc: "https://bsc-dataseed.binance.org" wallet: "${WALLET_ADDRESS}" xlayer: rpc: "https://rpc.xlayer.tech" wallet: "${WALLET_ADDRESS}"

Memory System

SEKI's persistent memory architecture enables true contextual awareness across sessions.

Architecture

The memory system operates on three tiers:

Working Memory

Current conversation context. Fast access, limited capacity. Automatically managed by the context window.

Episodic Memory

Stored as vector embeddings. Semantic search retrieves relevant past interactions, decisions, and outcomes.

Long-term Memory

Consolidated insights and learned patterns. Auto-curated from episodic memory through periodic reflection.

How It Works

// Store a memory await agent.memory.store({ content: "User prefers conservative DeFi strategies", type: "preference", importance: 0.8, tags: ["user", "defi", "risk"] }); // Semantic search const results = await agent.memory.search({ query: "What does the user think about yield farming?", limit: 5, threshold: 0.7 });

Auto-Consolidation

When enabled, the agent periodically reviews recent memories and distills them into long-term insights:

  • Identifies recurring patterns and preferences
  • Removes redundant or outdated entries
  • Generates summary memories with higher importance scores
  • Maintains a knowledge graph of entity relationships
Memory consolidation runs during idle periods to avoid impacting response times. Configure frequency via memory.consolidation_interval.

Self-Evolution

How SEKI agents continuously improve through autonomous learning loops.

The Evolution Loop

SEKI agents follow a continuous improvement cycle:

1. Observe

Monitor outcomes of every action. Track success rates, response quality, and user feedback signals.

2. Reflect

Periodically analyze performance patterns. Identify what works, what fails, and why.

3. Adapt

Generate new strategies, modify skill parameters, create new skills, or deprecate ineffective ones.

4. Validate

Test adaptations in sandbox. Only promote changes that demonstrate measurable improvement.

Evolution Types

  • Parameter Tuning — Adjusting thresholds, weights, and timing parameters based on outcome data
  • Strategy Mutation — Generating alternative approaches when current strategies underperform
  • Skill Synthesis — Combining existing skills to create new composite capabilities
  • Knowledge Distillation — Extracting reusable patterns from episodic memory into skills

Safety Controls

Evolution is sandboxed by default. All mutations are tested before deployment. Set evolution.auto_deploy: false to require manual approval for all changes.
evolution: enabled: true sandbox: true auto_deploy: false # Require approval max_mutations: 3 # Per cycle min_confidence: 0.85 # Deploy threshold

Architecture

SEKI's internal architecture: how the neural mesh, decision engine, and chain layer work together.

System Overview

SEKI is built as a layered architecture with clear separation of concerns:

Perception Layer

Ingests data from chains, APIs, and user inputs. Normalizes signals into a unified event stream for the decision engine.

Decision Engine

Core reasoning module. Combines rule-based logic with LLM inference. Prioritizes actions based on urgency, confidence, and resource availability.

Execution Layer

Translates decisions into actions: chain transactions, API calls, notifications, skill invocations. All actions are logged and auditable.

Memory Substrate

Persistent vector store underpinning all layers. Provides semantic retrieval, pattern matching, and temporal context to the decision engine.

Data Flow

┌──────────────────────────────────────────┐ │ External Data Sources │ │ Chains · APIs · Social · User Input │ └──────────────┬───────────────────────────┘ ▼ ┌──────────────────────────────────────────┐ │ Perception Layer │ │ Signal Normalization · Event Stream │ └──────────────┬───────────────────────────┘ ▼ ┌──────────────────────────────────────────┐ │ Decision Engine │ │ Rules → LLM → Skill Selection → Plan │ │ ↕ Memory Substrate │ └──────────────┬───────────────────────────┘ ▼ ┌──────────────────────────────────────────┐ │ Execution Layer │ │ Tx Signing · API Calls · Notifications │ └──────────────────────────────────────────┘

Key Design Principles

  • Event-Driven — All state changes flow through an event bus. Components are loosely coupled via pub/sub.
  • Sandbox-First — Skills execute in isolated environments. No skill can access system resources without explicit grants.
  • Audit Trail — Every decision and action is logged with full context. On-chain actions are verifiable via block explorers.
  • Graceful Degradation — If a data source is unavailable, the agent continues with reduced confidence rather than failing.

Writing Skills

A complete guide to creating custom skills for the SEKI platform.

Skill Anatomy

Every skill has three core files:

my-skill/ ├── SKILL.md # Human-readable description │ # The agent reads this to understand the skill ├── index.js # Executable logic └── config.yaml # Metadata, triggers, permissions

SKILL.md Format

# My Custom Skill ## Description What this skill does in 1-2 sentences. ## Triggers When to activate this skill: - User asks about X - Event Y is detected on chain - Scheduled every Z minutes ## Inputs - `token_address` — Contract address to analyze - `chain` — Target blockchain (bsc | xlayer) ## Outputs Returns a JSON object with analysis results. ## Examples > Analyze 0x1234 on BSC → Returns risk score, liquidity depth, holder distribution

config.yaml

name: "my-custom-skill" version: "1.0.0" description: "Brief description" triggers: intent: ["analyze_token", "check_risk"] schedule: "*/30 * * * *" events: ["large_transfer"] tools: ["web3", "coingecko"] permissions: ["chain:read", "memory:write"] evolution: mutable: true parameters: - name: "risk_threshold" range: [0.1, 0.9] default: 0.5

Agent Decision Logic

How SEKI agents make on-chain decisions: the complete reasoning pipeline.

Decision Pipeline

Every on-chain action flows through a multi-stage reasoning pipeline:

1. Data Ingestion

Collect real-time on-chain data: token prices, liquidity depth, holder distribution, contract events, gas prices.

2. Signal Processing

Normalize and score signals. Cross-reference with historical patterns from memory. Generate a weighted signal vector.

3. Risk Assessment

Evaluate potential outcomes against risk parameters. Check position sizing rules. Verify sufficient liquidity for exit.

4. LLM Reasoning

Present processed data to the LLM with full context. The model provides a structured decision with confidence score and rationale.

5. Execution

If confidence exceeds threshold: simulate transaction, verify gas, execute. Log the full decision trail to memory.

Confidence Thresholds

Action TypeMin ConfidenceApproval
Read-only query0.3Auto
Small trade (<1% portfolio)0.7Auto
Medium trade (1-5%)0.85Auto
Large trade (>5%)0.95Manual
Contract interaction0.9Manual
All thresholds are configurable. For production use, we recommend starting with manual approval for all trades and gradually increasing auto-approval limits as you build confidence in the agent's behavior.

Decision Logging

Every decision is stored with full context for audit:

Fund Safety

How SEKI protects user funds through multiple layers of security controls.

Security Architecture

Wallet Isolation

Agent wallets are isolated from user wallets. The agent operates with a dedicated hot wallet with configurable spending limits.

Transaction Simulation

Every transaction is simulated via eth_call before execution. Unexpected state changes trigger automatic abort.

Spending Limits

Per-transaction and daily spending caps. Breaching limits requires explicit user approval via secure channel.

Contract Allowlist

Agents can only interact with pre-approved contracts. New contract interactions require manual verification.

Safety Parameters

safety: max_per_tx: "100 USDT" max_daily: "1000 USDT" max_position: "5%" # of total portfolio slippage_max: "2%" gas_limit_multiplier: 1.5 require_simulation: true contract_allowlist: - "0x10ED...PancakeRouter" - "0xEf1c...SEKIStaking"
Never share agent wallet private keys. Use environment variables or a hardware wallet for signing. Consider a multisig for high-value operations.

Emergency Controls

  • Kill Switch — Instantly halt all agent operations via API or dashboard
  • Auto-Pause — Agent pauses if loss exceeds configurable threshold (default: 5% of portfolio)
  • Withdrawal Lock — Large withdrawals require time-locked confirmation (24h default)
  • Audit Log — Complete transaction history with decision context, exportable as CSV/JSON

FAQ

Frequently asked questions about the SEKI platform.

What is SEKI?

SEKI is an autonomous AI agent platform for Web3. It combines persistent memory, self-evolving skills, and on-chain execution to create intelligent agents that learn and improve over time.

Is SEKI safe to use with real funds?

SEKI has multiple safety layers: transaction simulation, spending limits, contract allowlists, and emergency kill switches. Start with small amounts and manual approval mode to build confidence.

What chains are supported?

Currently BSC and X Layer, with Ethereum, Polygon, and Arbitrum on the roadmap. The architecture supports any EVM-compatible chain.

How does the agent learn?

Through the observe-reflect-adapt evolution loop. The agent analyzes outcomes of its actions, identifies patterns, and generates improved strategies — all within a sandboxed environment.

Can I create custom skills?

Yes. Skills are modular and follow a simple structure (SKILL.md + index.js + config.yaml). See the Writing Skills guide.

What LLMs does SEKI support?

Any OpenAI-compatible API. GPT-4, Claude, Llama, Mistral — configure your preferred model in config.yaml.

Is there a token?

The SEKI token is used for governance, skill marketplace fees, and agent staking. Contract details are available on the project page.
