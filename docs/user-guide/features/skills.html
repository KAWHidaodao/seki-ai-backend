Skill Engine

Modular capabilities that Seki learns, improves, and shares autonomously.

What is a Skill?

A skill is a reusable, composable unit of agent behavior. Think of it as a specialized sub-routine that the agent can invoke when conditions are met. Each skill has:

  • Trigger Condition — When should this skill activate?
  • Execution Strategy — What does it do, and how?
  • Input/Output Schema — Typed data contracts
  • Performance Metrics — Success rate, avg profit, execution count
  • Confidence Score — Self-assessed reliability (0.0–1.0)

Skills are defined in YAML and can be auto-generated from successful task patterns by the evolution engine.

Skill Lifecycle

Every skill moves through a defined lifecycle. Promotion and demotion are automated based on performance data.

Skill States
  DRAFT ──→ TESTING ──→ LIVE ──→ DEPRECATED
    │          │          │
    │          │          └─ fitness < 0.3 for 48h
    │          └─ success_rate > 0.6 after 20+ runs
    └─ auto-generated or manually created
  • DRAFT — Newly created. Not executed. Available for review.
  • TESTING — Runs in sandbox mode with limited capital. Collecting performance data.
  • LIVE — Fully operational. Trusted for real execution.
  • DEPRECATED — Underperforming. Removed from active rotation. Kept for analysis.

Skill Definition

Skills are defined as YAML files in ~/.seki/skills/.

skills/meme-launch.yaml
name: meme-launch
version: 2.1.0
status: live
description: Deploy and promote meme tokens on BSC/Base

trigger:
  conditions:
    - social_buzz_score > 0.8
    - trending_topic_match: true
    - available_capital > 500 USDT
  cooldown: 4h

strategy:
  type: sequential
  steps:
    - deploy_token:
        chain: ${config.chain}
        supply: 1_000_000_000
        tax: { buy: 3, sell: 5 }
    - add_liquidity:
        pair: USDT
        amount: ${trigger.available_capital * 0.3}
    - promote:
        channels: [twitter, telegram]
        duration: 2h
    - monitor_exit:
        take_profit: 3x
        stop_loss: -40%

inputs:
  token_name: { type: string, required: true }
  token_symbol: { type: string, required: true }
  theme: { type: string, default: "auto" }

outputs:
  contract_address: address
  liquidity_tx: tx_hash
  pnl: float

constraints:
  max_capital: 2000 USDT
  max_concurrent: 1
  chains: [bsc, base]

meta:
  author: seki-core
  tags: [defi, meme, launch]
  confidence: 0.78
  total_runs: 47
  success_rate: 0.72

Auto-Generated Skills

When the evolution engine detects repeated successful patterns, it can synthesize a new skill automatically:

  1. Pattern Detection — Clusters of similar successful episodes are identified
  2. Template Extraction — Common triggers, actions, and parameters are abstracted
  3. Skill Generation — A YAML skill definition is auto-created in DRAFT state
  4. Sandbox Testing — The skill runs in TESTING mode for minimum 20 executions
  5. Promotion — If success_rate > 0.6, the skill promotes to LIVE
Discovery: Auto-generated skills appear in ~/.seki/skills/auto/. Review them with seki skills list --auto.

Skill Composition

Skills can be chained into pipelines. The output of one skill feeds into the next.

Pipeline Definition
pipeline: alpha-to-trade
steps:
  - skill: social-scan
    output_map: { trending_tokens → tokens }
  - skill: market-sense
    input_map: { tokens → targets }
    output_map: { signals → trade_signals }
  - skill: cex-trading
    input_map: { trade_signals → orders }
    filter: "confidence > 0.75"
