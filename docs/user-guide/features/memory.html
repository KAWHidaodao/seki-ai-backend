Memory System

How Seki remembers, learns, and builds context across thousands of observations.

Overview

Seki's memory is not a simple log. It's a multi-tier cognitive architecture inspired by human memory science. Four distinct layers work together to give the agent persistent, adaptive intelligence:

  • Working Memory — Current cycle context. The last 10 observations and active task state.
  • Episodic Memory — Stored experiences with vector embeddings. Searchable by semantic similarity.
  • Semantic Memory — Distilled knowledge: patterns, market models, and generalized rules extracted from episodes.
  • Meta-Memory — Memory about memory. Tracks which memories were useful, adjusts retrieval weights, and decides what to forget.

Memory Architecture

Working Memory

Working memory is the agent's immediate awareness — the scratchpad of the current decision cycle. It holds the most recent observations and expires after 3 cycles (~9 minutes).

What goes into working memory:

  • Price feeds from tracked tokens
  • Social signals (Twitter mentions, Telegram messages)
  • On-chain events (large transfers, contract interactions)
  • Active task context and pending decisions

  "timestamp": "2026-04-08T01:15:00Z",
  "observations": [

      "type": "price_feed",

Episodic Memory

When observations leave working memory, significant ones are encoded as episodes and stored in the vector database. Each episode captures the full context: what happened, what the agent did, and what the outcome was.

Storing an Episode
# Auto-stored after each decision cycle
episode = {
    "id": "ep_4821_a3f2",
    "timestamp": "2026-04-08T01:15:00Z",
    "context": {
        "market_state": "bull_micro",
        "volatility": "high",
        "sentiment": 0.82
    },
    "action": "buy_token",
    "params": {"token": "SEKI", "amount": "1000 USDT"},
    "outcome": {
        "profit_pct": 8.4,
        "timing_score": 0.91,
        "gas_efficiency": 0.76
    },
    "embedding": [0.12, -0.34, 0.56, ...]  # 1536-dim vector
}
Querying Episodic Memory
# Find similar past situations
results = seki.memory.query(
    context=current_observation,
    top_k=5,
    min_similarity=0.75,
    filters={"outcome.profit_pct": {">": 0}}
)

# Results: episodes where similar market conditions
# led to profitable outcomes
for ep in results:
    print(f"{ep.action} → {ep.outcome.profit_pct}%")

Semantic Memory

Raw episodes are noisy. Semantic memory is the distilled, generalized knowledge that emerges from pattern analysis across thousands of episodes.

Semantic Extraction
# Extracted pattern example
pattern = {
    "id": "pat_market_surge_buy",
    "description": "Volume surge + positive sentiment → buy signal",
    "conditions": {
        "volume_change_1h": "> 200%",
        "sentiment": "> 0.7",
        "whale_accumulation": true
    },
    "predicted_outcome": "price_increase_5_15pct",
    "confidence": 0.73,
    "sample_size": 142,
    "last_validated": "2026-04-07"
}

Semantic memory continuously updates as new episodes confirm or contradict existing patterns. Confidence scores rise or fall accordingly.

Meta-Memory

The meta-memory layer monitors the memory system itself. It answers: "Which memories are actually useful for making good decisions?"

Meta-Memory Feedback Loop
# After each decision, meta-memory evaluates:
feedback = {
    "decision_id": "dec_4821",
    "memories_retrieved": ["ep_4102", "ep_3987", "pat_surge_buy"],
    "memories_actually_useful": ["ep_4102", "pat_surge_buy"],
    "outcome_quality": 0.84,
    "adjustments": {
        "ep_4102": {"retrieval_weight": "+0.05"},
        "ep_3987": {"retrieval_weight": "-0.12"},
        "pat_surge_buy": {"retrieval_weight": "+0.08"}
    }
}

# Low-weight memories are candidates for pruning
seki.memory.auto_prune(threshold=0.15)

Memory Consolidation

Every 24 hours (configurable), the consolidation engine runs a full memory maintenance cycle:

  1. Episode Analysis — Cluster similar episodes, identify emerging patterns
  2. Pattern Extraction — Promote frequent episode clusters into semantic patterns
  3. Confidence Update — Re-validate all semantic patterns against recent data
  4. Pruning — Remove episodes older than retention period with low meta-memory scores
  5. Embedding Refresh — Re-encode semantic memories with updated embeddings
Resource Warning: Memory consolidation is CPU-intensive and queries the vector DB heavily. Schedule it during low-activity hours using consolidation_schedule: "0 4 * * *" (4 AM daily).
Monitor: Use seki memory stats to view memory utilization — episode count, semantic pattern count, vector DB size, and last consolidation time.
