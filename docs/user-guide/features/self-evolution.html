Self-Evolution

The core loop that makes Seki fundamentally different — an agent that improves itself through experience.

The Evolution Loop

Seki continuously runs a four-phase evolution cycle. Each cycle produces measurable improvements in strategy, timing, and risk management.

Evolution Cycle
    ┌──────────┐
    │ OBSERVE  │ ← Market data, social signals, on-chain events
    └────┬─────┘
         │
    ┌────▼─────┐
    │  LEARN   │ ← Pattern extraction, correlation analysis
    └────┬─────┘
         │
    ┌────▼─────┐
    │  EVOLVE  │ ← Strategy mutation, parameter tuning
    └────┬─────┘
         │
    ┌────▼─────┐
    │   ACT    │ ← Execute refined strategies
    └────┬─────┘
         │
         └──────→ Back to OBSERVE (continuous)

Phase 1: Observe

The observation phase ingests multi-modal data streams and normalizes them into a unified event format.

Data Sources

  • Market Data — Price feeds, order book depth, volume profiles, funding rates
  • Social Signals — Twitter mentions, Telegram activity, Discord sentiment
  • On-Chain Events — Large transfers, liquidity changes, contract deployments, wallet clustering
  • Agent Network — Insights shared by other Seki agents (peer learning)

Phase 3: Evolve

The evolution phase uses genetic-algorithm-inspired mutation to improve strategies. Only changes that improve the fitness score survive.

seki evolve --log --last 5
Generation 847 | Fitness: 0.73 → 0.78 (+6.8%)
├─ MUTATE  meme-launch: entry_threshold 0.85 → 0.79
├─ MUTATE  cex-trading: stop_loss 3.5% → 2.8%
├─ KEEP    market-sense: sentiment_weight 0.6
├─ PRUNE   old-arb-v2: fitness 0.31 (below threshold)
└─ SPAWN   hybrid-momentum: crossover(cex-trading, market-sense)

Generation 848 | Fitness: 0.78 → 0.81 (+3.8%)
├─ MUTATE  hybrid-momentum: lookback 12h → 8h
├─ KEEP    meme-launch: entry_threshold 0.79
└─ KEEP    cex-trading: stop_loss 2.8%

Mutation Types

  • Parameter Mutation — Adjust numerical parameters within defined bounds
  • Strategy Crossover — Combine successful elements from two strategies
  • Skill Acquisition — Discover and integrate new skills from the network
  • Pruning — Remove strategies that consistently underperform

Phase 4: Act

Evolved strategies are deployed with safety guardrails. Every action is sandboxed before live execution.

Execution Pipeline
Strategy Signal
     │
     ├─→ Risk Check ── position size, exposure limits
     │
     ├─→ Simulation ── paper trade against live data
     │
     ├─→ Approval ──── auto-approve or human-in-loop
     │
     └─→ Execute ───── tool dispatch + confirmation

Fitness Scoring

Each strategy is scored on multiple dimensions. The composite fitness score determines survival:

Fitness Function
fitness = (
  0.35 × profit_factor +
  0.25 × risk_adjusted_return +
  0.20 × win_rate +
  0.10 × execution_quality +
  0.10 × novelty_bonus
)

Survival threshold: 0.40
Elite threshold:    0.75 (protected from mutation)

Safety Guardrails

Critical: Evolution is bounded. Seki cannot evolve beyond configured risk parameters, maximum position sizes, or approved asset lists. The guardrails are immutable — even the evolution engine cannot modify them.
  • Max Drawdown Limit — Hard stop if portfolio drops below threshold
  • Position Sizing — Kelly criterion with configurable fraction
  • Asset Whitelist — Only approved tokens can be traded
  • Rate Limiting — Maximum trades per time window
  • Human Override — Emergency stop via CLI or Telegram command
