Tool Registry

External service integrations that give Seki hands to interact with the real world.

What is a Tool?

A tool is an authenticated connection to an external service — an exchange, a blockchain RPC, a social platform, or any API. While skills define what to do, tools define how to reach the outside world.

  • Authenticated — Each tool stores encrypted credentials
  • Rate-Limited — Built-in throttling respects API limits
  • Health-Checked — Continuous connectivity monitoring
  • Hot-Swappable — Replace or upgrade without restarting the agent

Registering a Tool

Tools are registered via the CLI or configuration file. Credentials are encrypted at rest using AES-256-GCM.

CLI Registration
# Interactive setup with credential prompts
seki tools add binance

# From environment variables
BINANCE_API_KEY=xxx BINANCE_SECRET=yyy seki tools add binance --from-env

# From config file
seki tools add --config tools/binance.yaml

Tool Configuration

tools/binance.yaml
name: binance
type: exchange
version: 1.0.0

connection:
  rest_url: https://api.binance.com
  ws_url: wss://stream.binance.com:9443
  timeout: 10s
  retry: 3

auth:
  type: hmac-sha256
  credentials:
    api_key: ${env.BINANCE_API_KEY}
    secret: ${env.BINANCE_SECRET}

rate_limits:
  orders: 10/s
  weight: 1200/min
  websocket: 5 connections

permissions:
  - spot_trade
  - futures_trade
  - read_balance
  deny:
  - withdrawal
  - transfer

Permission Model

Tools use a least-privilege permission model. Each skill must declare which tool capabilities it needs, and the agent validates permissions before execution.

Permission Check Flow
Skill Request ──→ Permission Validator ──→ Tool Executor
                        │
                  ┌─────┴─────┐
                  │ ALLOWED?  │
                  ├───────────┤
                  │ YES → Execute
                  │ NO  → Deny + Log
                  └───────────┘
  • Explicit Allow — Only declared permissions are granted
  • Deny Override — Critical operations (withdrawal, transfer) require manual approval
  • Audit Trail — Every tool invocation is logged with caller, params, and result

Tools are continuously monitored. If a tool becomes unhealthy, the agent automatically falls back to alternatives or pauses dependent skills.

Custom Tool Development

Create custom tools by implementing the ToolAdapter interface:

TypeScript
import { ToolAdapter, ToolContext, ToolResult } from '@seki/core';

export class CustomExchangeTool implements ToolAdapter {
  name = 'my-exchange';
  version = '1.0.0';

  async connect(ctx: ToolContext): Promise<void> {
    // Initialize connection
    this.client = new ExchangeClient(ctx.credentials);
    await this.client.ping();
  }

  async execute(action: string, params: Record<string, any>): Promise<ToolResult> {
    switch (action) {
      case 'place_order':
        return this.placeOrder(params);
      case 'get_balance':
        return this.getBalance(params);
      default:
        throw new Error(`Unknown action: ${action}`);
    }
  }

  async healthCheck(): Promise<boolean> {
    return this.client.ping();
  }

  async disconnect(): Promise<void> {
    await this.client.close();
  }
}
Plugin Directory: Custom tools are loaded from ~/.seki/tools/. Each tool is a directory with an index.ts entry point and a tool.yaml manifest.
