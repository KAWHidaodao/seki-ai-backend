Configuration

Fine-tune your Seki Agent. Environment variables, config files, and runtime settings.

Config File

The primary configuration lives at ~/.seki/config.yaml. This file is created by seki init and controls every aspect of agent behavior.

~/.seki/config.yaml
# === Chain & Network ===
chain: bsc                          # bsc | base | xlayer
rpc_url: https://bsc-dataseed1.binance.org

# === Wallet ===
wallet:
  private_key: ${SEKI_PRIVATE_KEY}   # always use env vars!

# === Persona ===
persona: hunter                      # hunter | strategist | herald

# === Memory ===
memory:
  provider: chroma                   # chroma | pinecone | weaviate
  max_episodes: 1000
  consolidation_interval: 24h

# === Skills ===
skills:
  auto_discover: true
  marketplace_url: https://skills.seki-ai.com

# === Evolution ===
evolution:
  enabled: true
  reflection_interval: 30m
  mutation_rate: 0.15
  min_confidence: 0.7
Validate: Run seki config validate to check your configuration for errors before starting the agent.

Environment Variables

Sensitive values should be set as environment variables rather than stored in config files directly.

.env
# Core
SEKI_CHAIN=bsc
SEKI_RPC_URL=https://bsc-dataseed1.binance.org
SEKI_PRIVATE_KEY=0x...

# Persona
SEKI_PERSONA=hunter

# AI Provider
SEKI_OPENAI_KEY=sk-...

# CEX Integration
SEKI_BINANCE_KEY=...
SEKI_BINANCE_SECRET=...

# Feature Flags
SEKI_EVOLUTION=true

Environment variables override config.yaml values. Load them via seki start --env-file .env or export them in your shell.

Memory Configuration

The memory subsystem supports multiple vector database providers. Chroma is the default (local, zero-config). For production workloads, consider Pinecone or Weaviate.

Memory — Chroma (default)
memory:
  provider: chroma
  path: ~/.seki/memory/chroma
  max_episodes: 1000
  embedding_model: text-embedding-3-small
  consolidation_interval: 24h
Memory — Pinecone
memory:
  provider: pinecone
  api_key: ${PINECONE_API_KEY}
  environment: us-east-1
  index: seki-memory
  embedding_model: text-embedding-3-small
Memory — Weaviate
memory:
  provider: weaviate
  url: http://localhost:8080
  class_name: SekiEpisode
  embedding_model: text-embedding-3-small

Evolution Settings

Control how aggressively the agent evolves its strategies.

Evolution Config
evolution:
  enabled: true
  reflection_interval: 30m     # how often the meta-agent reflects
  mutation_rate: 0.15           # probability of strategy mutation
  min_confidence: 0.7           # minimum confidence to execute
  max_active_strategies: 10     # concurrent strategy cap
  auto_prune: true              # remove underperforming strategies
  prune_threshold: 0.3          # prune if fitness drops below this
  • reflection_interval — How often the meta-agent reviews recent performance. Shorter = more adaptive, higher compute cost.
  • mutation_rate — Probability a strategy parameter is randomly adjusted during evolution. Range: 0.0–1.0.
  • min_confidence — The agent won't execute a strategy below this confidence threshold.
  • max_active_strategies — Hard cap on concurrent live strategies to limit resource usage.
  • auto_prune — When enabled, strategies with fitness below prune_threshold are automatically deprecated.

Security

Never commit private keys to version control. Always use environment variables or .env files for sensitive data.

Create a .gitignore to protect sensitive files:

.gitignore
.env
*.key
~/.seki/config.yaml
node_modules/

Additional security recommendations:

  • Use a dedicated wallet for the agent — never your main wallet
  • Start with sandbox: true for CEX tools to use testnet
  • Set max_trade_size limits in tool configuration
  • Enable human_approval: true for trades above a threshold
  • Rotate API keys regularly
